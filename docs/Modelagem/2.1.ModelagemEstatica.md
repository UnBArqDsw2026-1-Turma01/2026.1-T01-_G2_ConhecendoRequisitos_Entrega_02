## Escolha dos diagramas

Os diagramas desta entrega foram escolhidos para representar a arquitetura estática do sistema em níveis complementares de abstração:

- **Diagrama de Classes:** foi selecionado para detalhar as entidades do domínio, seus atributos, métodos e relacionamentos. Essa escolha permitiu transformar os requisitos funcionais em uma estrutura orientada a objetos, servindo como base para implementação e para validação das responsabilidades de cada classe.
- **Diagrama de Componentes:** foi adotado para mostrar a organização macro do sistema, evidenciando subsistemas, serviços e dependências. A equipe utilizou esse artefato para visualizar como as partes principais da aplicação se conectam, facilitando decisões arquiteturais e de modularização.
- **Diagrama de Implantação:** foi incluído para representar a visão de infraestrutura, destacando onde os componentes seriam executados e como ocorreria a comunicação entre nós de execução. Essa escolha foi importante para antecipar necessidades de ambiente e apoiar a viabilidade técnica da solução.

Em conjunto, os três diagramas cobrem a modelagem estática desde o nível conceitual de classes até a visão arquitetural e de execução, oferecendo uma documentação mais completa do sistema.
